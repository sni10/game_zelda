# 🚀 Release 0.3.0 — Combat Loop, Enemies & Architecture Cleanup

## 📋 Описание релиза

Версия **0.3.0** — это поворотный релиз. Игра впервые превращается в нечто
**играбельное по циклу «исследую → агрю → бью → собираю дроп»**: появилась
полноценная система оружия, четыре типа врагов, авто-респавн, подбираемый
лут и двухслойные карты с холмами/крышами. Параллельно проведён большой
архитектурный рефакторинг по принципам **SOLID/SRP** — каждый «жирный»
файл разнесён на специализированные классы.

> Дата релиза: 26 апреля 2026
> Тип релиза: Minor (новая функциональность + рефакторинг без breaking API)
> Совместимость сохранений: **несовместимо** с v0.2.1 (см. Known Issues)

## ✨ Новые игровые механики

### ⚔️ Система оружия (4 типа)
- **4 класса оружия** через ABC `Weapon` + конкретные подклассы (`Sword`, `Spear`, `Bow`, `Hammer`)
- Переключение по клавишам **1..4**
- **Симметричные зоны атаки**: каждое оружие задаёт свой `reach`, `damage`, цвет и форму поражения для всех 8 направлений (включая диагонали)
- Цветная индикация активной зоны атаки (`weapon.color`)

### 👹 Враги, AI и спавн
- **Несколько типов врагов** с конфигурируемыми параметрами в `config.ini` (HP, скорость, урон, цвет, зона патруля)
- **Аксиальный патруль**: враг движется по своей оси, разворачиваясь у границ зоны
- **Контактный урон**: касание врага → урон игроку с уважением i-frames
- **Авто-респавн**: при удалении игрока за пределы зоны враг возрождается, поддерживая «живой» мир
- Учёт `kills` в статистике + анимация смерти

### 🎁 Система пикапов (`PickupManager`)
- Враги могут дропать **монеты, аптечки, XP**
- Подбор по проходу игрока через зону пикапа
- Балансировка дропа в `config.ini`
- HUD отображает уровень, XP-бар и количество монет

### 🌄 Двухслойные карты (Z=0 + Z=2 overlay)
- ASCII-карты теперь поддерживают **парный overlay-файл** (`<world>_overlay.txt`)
- Тип тайла **`HILL_SURFACE` (`H`)**:
  - На земле — непроходимая стенка холма
  - На overlay — **полупрозрачная крыша**, рисуется поверх игрока, когда тот проходит «под холмом»
- Реализовано через `world.draw_overlay(screen, player_rect)`

### 🐰 Burrow-механика
- Прототип системы «нор» (туннели/проходы): анимация спуска/подъёма игрока, временная подмена движения

### 🏃 Спринт
- Удержание **Shift** → множитель скорости (`PLAYER_SPRINT_MULTIPLIER` в конфиге)
- Учитывается в `Player.handle_input()` и взаимодействует с модификаторами поверхности

### 📈 Прогрессия игрока
- Уровни, XP, монеты, бонус к урону (`damage_bonus`) — всё в `PlayerStats`
- Формула XP до следующего уровня: `base * (level ** growth)` (настраивается в config)
- Опциональное полное лечение при level-up

## 🧱 Архитектурный рефакторинг (SOLID/SRP)

> Подробности — в `REFACTOR_PLAN.md`. Все 156 тестов остались зелёными,
> публичный API сохранён через `@property`-делегирование.

| Было | Стало | Профит |
|------|-------|--------|
| `core/game.py` 420 LOC (God Object: цикл + HUD + логи) | `game.py` ~270 + `ui/hud.py` + `utils/session_logger.py` | Игровой цикл больше не знает про шрифты HUD и файлы логов |
| `world/world.py` со встроенной камерой | + `world/camera.py` (чистая математика без pygame) | Камера юнит-тестируется без окна |
| `world/terrain.py` (модель + I/O в одном файле) | `terrain.py` (модель тайла) + `world/map_loader.py` (парсинг + I/O) | Сменить формат карт можно не трогая рендер |
| `entities/player.py` 287 LOC (HP + ввод + бой + рендер) | `player.py` (оркестратор) + `player_stats.py` + `player_combat.py` | HP и боёвка тестируются изолированно |

**Принципы, которые соблюдаются:**
- **SRP** — у каждого класса одна ответственность
- **Composition over Inheritance** — `Player` собирается из `PlayerStats` + `PlayerCombat`, оружия используют ABC
- **Configuration-First** — никаких magic numbers в коде, всё в `config.ini` через `get_config()`/`get_color()`

## 🗺️ Упрощение архитектуры мира

В рамках экспериментов до v0.3.0 был внедрён ECS + multi-world (main_world / cave_world / underground_world) с порталами и Z-уровнями. **Это было переинжинирено для текущего этапа MVP** и было сознательно удалено в коммите `feat(game): simplify game structure by removing multiworld mechanics`:
- Один активный мир
- Прямой `Player ↔ World` без слоя синхронизации ECS
- Карты по-прежнему ASCII, но без необходимости держать 3 контекста в памяти

> Это решение задокументировано в `AGENTS.md` и `CLAUDE.md`. Multi-world вернётся отдельной веткой, когда базовые механики (комбат, инвентарь, прогрессия) будут стабильны.

## 🐛 Known Issues — что чинится в v0.3.1

### 1. Меню «Загрузить игру» — заглушка
Пункт меню `load_game` в `core/game.py` запускает **новую игру** вместо выбора слота:
```python
elif action == "load_game":
    print("Load game - функция будет реализована позже")
    self.start_new_game()  # ← BUG: затирает прогресс
```
**Workaround:** использовать `F9` (quickload) вместо пункта меню.

### 2. F9 / «Продолжить игру» — частичное восстановление
Состояние восстанавливается **не полностью**:
- ✅ Позиция игрока, HP, направление взгляда
- ❌ Текущее оружие и его индекс — всегда сбрасывается на дефолт
- ❌ Живые враги — после загрузки спавнятся заново, прогресс боя теряется
- ❌ Лежащие на земле пикапы (монеты, аптечки)
- ❌ `GameStats` (убийства, дистанция, время) — обнуляются
- ⚠️ Мёртвая строка в `save_system.py:138` — артефакт после рефакторинга `Player → PlayerStats`

### 3. Нет UI выбора слотов сохранения
- Существует только один файл `saves/quicksave.json`
- Нет таймстемпов, превью, удаления слотов

### 4. Нет автосохранений
- При случайном выходе/смерти прогресс с момента последнего F5 теряется
