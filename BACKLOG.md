### 🟡 v0.3.2 — UI слотов

**Два типа сохранений (раздельно, не пересекаются):**
- **Quicksave (F5/F9)** — один отдельный постоянно перезаписываемый файл (`quicksave.json`). Без подтверждений, без меню. Текущее поведение F5/F9 сохраняется как есть.
- **Manual saves (F6 + меню)** — слоты, лимит **10**. Создаются и загружаются только через меню. Хранятся отдельной папкой/префиксом, не смешиваются с quicksave.

**Реализация:**
1. `SaveSystem.list_saves()` — список **manual-слотов** с метаданными (timestamp, level, playtime, hp). Quicksave в этот список не входит.
2. Новый класс `src/ui/load_menu.py` — список слотов с навигацией ↑↓, Enter, Del, Esc.
3. Новые состояния `GameState.LOAD_MENU` (загрузка) и `GameState.SAVE_MENU` (сохранение).
4. **F5 = quicksave** (как сейчас, перезаписывает `quicksave.json`); **F9 = quickload** (как сейчас).
5. **F6 = open save menu** (выбор manual-слота для сохранения); из главного меню «Загрузить игру» открывается LOAD_MENU со списком всех manual-слотов **+ отдельной строкой quicksave сверху**.
6. **Лимит — 10 manual-слотов.** Попытка сохранения при заполненном списке → принудительный выбор слота для перезаписи (нет «авто-вытеснения старых»).
7. **Подтверждение перезаписи:** при сохранении в занятый manual-слот — модальное окно «Перезаписать сохранение от {timestamp}, уровень {N}? Y/N». Quicksave перезаписывается без подтверждения.
8. **Подтверждение удаления:** Del в меню → «Удалить сохранение? Y/N». Quicksave удалить нельзя.
9. Превью миникарты в слоте (опц.)
