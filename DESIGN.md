# 🎮 DESIGN — направление и подход

Живой документ: куда движется игра, какие решения уже приняты, чего НЕ делаем.
Обновлять при каждом значимом решении по геймдизайну или архитектуре.

---

## 🎯 Что мы делаем

**Top-down shooter + melee, ARPG-механики, готический космос-хоррор-сеттинг.**

**Жанр:** action-shooter с прокачкой, дропом и экипировкой. Сеттинг и эстетика **вдохновлены Warhammer 40k** (морпех в готическом будущем vs ксеносы), но это **не клон** — свои механики, свой мир, **никаких trademark-терминов в коде**.

**Игровой цикл:**
1. Войти в зону → отбиться от тварей → собрать дроп (патроны / оружие / броню)
2. Прокачать персонажа и оружие → вернуться в безопасную зону / магазин
3. Двинуться дальше / в следующую локацию

**Аналоги по геймплею:**
- **Crimsonland** — top-down shooter с прокачкой и пушками
- **Alien Breed** — top-down с морпехом и инопланетной угрозой
- **Helldivers (top-down)** — кооп-шутер с тяжёлыми орудиями
- **Diablo** — лут, экипировка в слотах, аффиксы

**Чего НЕ делаем:**
- ❌ **Не RTS** (StarCraft, Warcraft) — выделение групп, постройки, fog of war. Другой движок.
- ❌ **Не Zelda-like** — пробовали, ушли. Текущая база подходит лучше под shooter+ARPG.
- ❌ **Не лёгкий cartoon** (Утиные истории и т.п.) — сеттинг серьёзный/брутальный.
- ❌ **Не повторяем `docs/scenario/`** — старые документы устарели до фиксации жанра.
- ❌ **Не используем trademarked-термины Warhammer** в коде (`Space Marine`, `Bolter`, `Adeptus Astartes`, `Imperium`). Только generic: `Marine`, `Trooper`, `Pulse Rifle`, `Grenade Launcher`, `Xeno`.

## 🧭 Принципы разработки

1. **Механики раньше сюжета.** Сначала играбельный билд, потом сценарий поверх него.
2. **Прототипы раньше архитектуры.** Не масштабируй то, чего ещё нет в игре.
3. **Data-driven раньше классов.** Когда контента много — данные в файлах, не классы для каждой единицы.
4. **Не делать преждевременную абстракцию.** Главный убийца pet-проектов.
5. **Эстетика — последний слой.** Сначала числа и поведение, потом спрайты и звук.

## 🎖️ Боевая модель

### 4 категории оружия

| Категория | Тип | Боезапас | Generic-имя | Концепт |
|---|---|---|---|---|
| **Короткий меч** | melee, ближний | нет | `ShortBlade` | быстрый, ближний контакт |
| **Длинный меч** | melee, средняя дистанция | нет | `LongBlade` | медленный, больший reach |
| **Ствол** | ranged, прямой огонь | патроны | `PulseRifle` / `AutoRifle` | основное стрелковое |
| **Гранатомёт** | ranged AoE | гранаты | `GrenadeLauncher` | по площади, тяжёлый |

Сейчас `src/entities/weapons.py` уже содержит 4 базовых класса (Strategy pattern). При v0.4.x они переименуются под новый сеттинг и расширятся параметрами.

### Дуальное владение (Dual-wield)

Игрок носит **2 оружия одновременно** — `main_hand` / `off_hand`:

- ✅ **Меч + меч** — любая комбинация (короткий+длинный, два коротких, два длинных)
- ✅ **Ствол + ствол** — любая комбинация
- ⚠️ **Гранатомёт ВСЕГДА в паре с пулемётом/стволом** — это инвариант системы. Два гранатомёта запрещены.
- ❓ **Меч + ствол** — открытый вопрос (см. ниже)

Управление будет уточнено в прототипе v0.4.0 (ЛКМ/ПКМ или 1/2-переключение).

### Боеприпасы

- Меч / длинный меч — бесконечны (melee).
- Ствол — счётчик патронов, дроп с врагов / сундуков.
- Гранатомёт — счётчик гранат, реже патронов.

### Броня — 4 слота

| Слот | Что даёт |
|---|---|
| **Шлем** (`helmet`) | защита головы, бонус к видимости/HUD |
| **Нагрудник** (`chest`) | основная защита HP |
| **Поножи** (`legs`) | защита + скорость |
| **Наручи** (`arms`) | бонус к атаке / парирование |

Каждый предмет брони — слот + базовые статы + (опц.) аффиксы.

### Лут и сундуки

- Враги дропают: HP / монеты / XP (как сейчас) **+ изредка оружие или броню**.
- На карте — **сундуки** (новый объект). Открываются клавишей `E`. Гарантированный лут с шансом на редкость.
- **Редкость:** common / uncommon / rare. Влияет на статы и количество аффиксов.

### Прокачка — двухслойная

1. **Уровень персонажа** (есть сейчас) — общий XP, рост max HP.
2. **Прокачка оружия по категориям** (новое) — XP отдельно по `short_blade` / `long_blade` / `rifle` / `grenade_launcher`. Использование = опыт. Открывает перки:
   - **Зажигательные патроны** (DoT для ствола)
   - **Тройные гранаты** (×3 области для гранатомёта)
   - **Парирование** (короткий меч)
   - **Кровотечение** (длинный меч)
   - и т.д.

Появится в v0.5.x когда оружие переедет на data-driven.

## 📐 Подход к масштабированию контента

### Где мы сейчас (v0.3.x)

| Контент | Кол-во | Паттерн |
|---|---|---|
| Оружие | 4 категории × 1 вариант | **Strategy** в `src/entities/weapons.py` |
| Враги | 3 (Light / Heavy / Fast) | Классы + Factory |
| Пикапы | 3 (Heart / Coin / XP) | Классы |
| Броня | 0 | — |

### Куда движемся (v0.4.x — v0.5.x)

| Контент | Будет | Паттерн |
|---|---|---|
| Оружие | 4 категории × 5+ вариантов = **20+** | **data-driven** YAML + tags |
| Броня | 4 слота × 5+ тиров = **20+** | **data-driven** YAML + tags |
| Аффиксы | **10+** модификаторов | Affix-список на предмете (decorator-через-данные) |
| Враги | 5–8 типов + мини-боссы | Расширение Factory |
| Пикапы | 3 (HP/coin/XP) + ammo + weapon + armor | Класс на категорию |

### Порядок миграции

1. **v0.4.0** — добавляем боеприпасы и дуальное владение **в текущей Strategy-модели**. 4 оружия → 4 оружия, но богаче.
2. **v0.4.1** — добавляем броню (новая система с нуля → сразу data-driven).
3. **v0.4.2** — добавляем лут и сундуки (расширение `Pickup`).
4. **v0.4.3** — добавляем инвентарь (UI, экран персонажа).
5. **v0.5.0** — **миграция оружия на data-driven** + аффиксы. Это и есть тот самый порог «8+ оружий» из принципов.
6. **v0.5.1** — перки и прокачка оружия по категориям.

### Skyrim-style: один класс + теги

Один `Weapon`/`Armor`, всё остальное — **данные** + **теги**:

```yaml
# data/weapons.yaml
short_blade_iron:
  category: short_blade
  damage: 1
  speed: 1.2
  tags: [melee, blade, iron, short]

long_blade_power:
  category: long_blade
  damage: 3
  speed: 0.9
  tags: [melee, blade, power, long]

rifle_standard:
  category: rifle
  damage: 1
  ammo_type: bullets
  rate_of_fire: 5.0
  tags: [ranged, rifle, standard]

rifle_incendiary:
  category: rifle
  base: rifle_standard
  tags: [ranged, rifle, fire]
  affixes: [incendiary]
```

Подсистемы (магазин, инвентарь, перки) реагируют **на теги**, не на типы:
- «Перк +20% урон клинками» → фильтр `blade`
- «Только морпех в power-броне» → фильтр `power`

### Diablo-style: affix-система поверх

```python
weapon.affixes = [
    Affix("incendiary", on_hit_dmg_over_time=2, fire_dmg=1),
    Affix("triple_grenades", aoe_multiplier=3.0),  # для гранатомёта
]
```

Каждый аффикс — данные + (опц.) callback в системе урона. Decorator pattern, но через данные а не классы.

### ECS — пока не трогаем

В `src/core/ecs/` остался скелет от удалённой multi-world системы. **Не использовать.** Текущей модели Player + Enemy + Pickup + Weapon (Strategy) хватает на всё что планируется до v1.0. Повторно посмотрим только если игра пойдёт в сторону кооп / много типов NPC.

## 🚫 Антипаттерны для этого проекта

- **Класс на каждую единицу контента** (`IronShortBlade(Weapon)`, `SteelShortBlade(Weapon)`, ...) — после 5–10 единиц превращается в ад. Сразу делать data-driven.
- **«Универсальный» Item-класс с 50 полями** — другая крайность. Базовый класс + теги + данные.
- **Глубокая иерархия наследования** (`Weapon → MeleeWeapon → ShortBlade → PowerShortBlade → IncendiaryPowerShortBlade`). Заменять композицией / тегами / аффиксами.
- **GoF паттерны "ради паттернов"**. В геймдеве работают: Strategy, Factory, Component composition, Observer (события). Остальные — по ситуации.
- **Trademark-термины в коде**: `BolterWeapon`, `SpaceMarinePlayer`, `AdeptusAstartesArmor`, `ImperiumGuard` — нельзя. Только generic.
